枯木 · 2020-10-9 · 次阅读

1. 数据的存储方式

​ 特定的文件 / 内存 / 第三方云服务器 / 数据库服务器

2. 数据库

​ 在存储数据的时候，按照特定的形式来组织存储数据，目的是为了对数据的操作——增删改查

  1. 数据库的发展历史

    网状数据库——层次型数据库——关系型数据库——非关系型数据库(NoSQL)

  2. 关系型数据库逻辑结构

    Server -> Database -> Table -> Row -> Column
数据库服务器   数据库    数据表     行      列

  3. 常见的关系型数据库

    1. SQLite

      微型数据库，常用于移动设备。

    2. MySQL

      开源中小型数据库，可用于各种操作系统。

    3. PostgreSQL

      开源中小型数据库。

    4. SQL Server

      Microsoft开发的中小型数据库，只用于Windows系统。

    5. Oracle

      Oracle公司开发的中大型数据库，可用于各种操作系统。

    6. DB2

      IBM公司开发的中大型数据库，常与IBM服务器搭配。

3. Mysql

​ 软件： Xampp

  1. 一个MySQL数据库系统分为如下两个部分：

    • 服务器端：负责存储和管理数据
    • 客户端：用于向服务器发出增删改查操作命令

  2. mysql部署结构

    1. 服务器端：启动服务 端口3306

      ​ C:/xampp/mysql/bin/mysqld.exe

    2. 客户端：链接服务器

      ​ C:/xampp/mysql/bin/mysql.exe

  3. 使用客户端链接服务器端

    mysql.exe -h127.0.0.1 -P3306 -uroot -p

    -h host 主机，服务器的地址或域名

    -P port 端口，

    -u user 用户，root(默认管理员用户)

    -p password 密码，xampp下root的密码为空

    简写形式：mysql -uroot

  4. mysql常用管理命令

    quit； 推出服务器的链接

    show databases; 显示当前数据库服务器下所有的数据库

    use 数据库名称； 进入指定的数据库

    show tables; 查看数据库中所有的表

    desc 表名；描述表的结构

    show create table 表名； 查看创建表所用的命令、

  5. SQL 命令

    结构化查询语言，主要用于操作关系型数据库，包含有对数据的操作——增删改查

    1. 交互模式

      在客户端命令行输入一行回车，服务器端就会执行一行，适用于临时性的查看数据

    2. 脚本模式

      客户端把要执行的命令写在一个脚本文件中，然后一次性的提交给服务器执行，适用于批量的操作数据在连接的同时提交脚本

      语法：mysql -uroot<脚本路径

  6. SQL命令的语法规范

    1. 一行SQL命令可以跨越多行，以英文的分号作为结束
    2. SQL命令中不区分大小写，习惯上关键字大写，非关键字小写
    3. 假设某一条SQL命令出现语法错误，则此条语句及后面的所有语句不再执行
    4. 分为单行注释（#）和多行注释（/ /）
